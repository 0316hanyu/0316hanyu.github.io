String、StringBuffer、StringBiulder的区别

在开发中，我们无时无刻都会与字符串打交道，本文主要对Java中的String、StringBuffer、StringBiulder，进行一个比较。

String

首先我们来看一下源码：

由以上的代码可以看出，在Java中String类其实就是对字符数组的封装。因为都是由关键字 final 修饰的，所以我们可以知道 String 是不可变的，我们一旦初始化了，就会在常量池永久的生成字符数组（常量池没有的情况下）。

下面我们谈一下 String 不可变的好处：

  • String 作为我们常用的 HashMap 的key，其 hash 也只需要计算一次
  • String 常用作与网络参数、文件名等操作，保证安全性
  • String 在多线程操作中安全，不会影响我们的功能

但是 String 在一些逻辑中又是不适合的：
如当我们需要大量字符串的操作（拼接、修改、赋值等），会不断的在常量池中创建，造成空间的浪费，且效率较低。

StringBuffer 与 StringBiulder

首先 二者是Java提供的字符串操作类，在对字符串操作较多的场景中被广泛使用，其底层也是使用 char 数组进行存储，只不过StringBuffer 与 StringBiulder 会进行扩容操作，并且提供了很多方法，使用起来非常方便、高效。

StringBuffer 与 StringBiulder 的区别

StringBuffer 在方法上 加上了 synchronized 锁，保证了线程安全。相反 StringBiulder 是线程不安全的。

小结

（1）如果要操作少量的数据用 String；

（2）多线程操作字符串缓冲区下操作大量数据 StringBuffer；

（3）单线程操作字符串缓冲区下操作大量数据 StringBuilder。
